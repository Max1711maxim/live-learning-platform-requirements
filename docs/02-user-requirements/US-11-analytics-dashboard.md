# US-11: Дашборд аналитики продаж и активности

**Идентификатор:** US-11

**Роли:** Преподаватель (Автор курса)

**Формулировка требования:**
Как преподаватель, я хочу видеть на одном экране графики выручки, конверсии в покупку, количество активных студентов и самые просматриваемые уроки, чтобы оценивать окупаемость курса и корректировать контент на основе данных.

**Атрибуты:**

| Атрибут | Значение |
|:---|:---|
| Источник | Impact Mapping, Модель Кано (One-dimensional) |
| Приоритет | Высокий (One-dimensional) |
| Стабильность | Стабильное |

**Связанные требования:**
- NFR-01: Данные на дашборде должны обновляться не реже чем раз в 60 секунд в режиме реального времени.
- NFR-04: Доступ к дашборду аналитики должен предоставляться только автору курса; передача агрегированных данных между сервисами должна осуществляться по защищённому протоколу (TLS 1.3).
- NFR-05: Система должна обеспечивать отображение дашборда без деградации производительности при количестве студентов курса до 10 000.
