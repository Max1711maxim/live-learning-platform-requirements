# NFR-04: Безопасность данных

**Идентификатор:** NFR-04

**Формулировка требования:**
Система должна обеспечивать следующий уровень безопасности:

1. **Передача данных:** все соединения между клиентом и сервером, а также между внутренними сервисами должны использовать TLS 1.3.
2. **Платёжные данные:** обработка, передача и хранение платёжной информации должны соответствовать стандарту PCI DSS.
3. **Персональные данные:** хранение и обработка должны соответствовать требованиям 152-ФЗ (для РФ) или GDPR (для пользователей из ЕС).
4. **Аутентификация:** обязательная двухфакторная аутентификация для преподавателей при входе в дашборд аналитики и проведении вебинаров.
5. **Авторизация:** доступ к дашборду аналитики и результатам студентов должен предоставляться только автору соответствующего курса.
6. **Видеоконтент:** доступ к видеоурокам должен предоставляться только по подписанным URL с ограниченным сроком действия (token-based access).
7. **Офлайн-материалы:** скачанные видео и PDF должны быть зашифрованы и доступны только внутри приложения платформы.

**Атрибуты:**

| Атрибут | Значение |
|:---|:---|
| Источник | US-05 (Возврат средств), US-11 (Дашборд), Бизнес-риски |
| Приоритет | Критический |
| Стабильность | Стабильное |

**Связанные требования:**
- US-05: Запрос возврата средств.
- US-11: Дашборд аналитики.
- FR-01: Блокировка возврата при невыполнении условий.
