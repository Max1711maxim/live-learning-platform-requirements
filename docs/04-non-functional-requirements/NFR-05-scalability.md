# NFR-05: Масштабируемость

**Идентификатор:** NFR-05

**Формулировка требования:**
Архитектура системы должна обеспечивать:

1. **Горизонтальное масштабирование:** возможность увеличения количества серверов приложений, баз данных и CDN-узлов без остановки работы системы.
2. **Одновременные вебинары:** поддержка проведения не менее 50 параллельных вебинаров с суммарной аудиторией до 5000 зрителей.
3. **Пиковые нагрузки:** система должна выдерживать рост нагрузки в 3 раза на период запусков популярных курсов без деградации времени отклика (NFR-01).
4. **Дашборд аналитики:** отображение данных без снижения производительности при количестве студентов курса до 10 000.
5. **Генерация сертификатов:** при пиковой нагрузке система должна ставить задачи генерации PDF в очередь и обрабатывать их асинхронно (не более 5 минут ожидания).
6. **Объём хранилища:** возможность хранения до 100 ТБ видеоконтента с автоматической архивацией неактивных курсов.

**Атрибуты:**

| Атрибут | Значение |
|:---|:---|
| Источник | Бизнес-цели (5000+ студентов, 100+ преподавателей) |
| Приоритет | Высокий |
| Стабильность | Стабильное |

**Связанные требования:**
- FR-06: Автоматическая генерация и выдача сертификата.
- NFR-01: Время отклика интерфейса.
- NFR-02: Доступность сервиса вебинаров.
